《通信软件开发与应用》课程结业报告

网页概述

这次期末结业我采用静态的框架，做了一个关于狗的一个网站，并且将他托管在了gitgub上。由于我建立的是静态网页，所以我采用了MDB模板。MDB全名为MDBootstrap，Material Design 是谷歌于2014年推出用于媲美苹果的视觉设计规范。请访问其Material Design 官网了解。以Bootstrap 为基础，同时遵循 MD 规范的靓丽 CSS 框架也不少，构建了许多直接使用的组件。其中以 MDBootstrap 为最佳它有丰富的CSS模板，JS模板，所以只需要进行HTML和CSS的相应改变即可。 基于MDB的框架，我们可以快速的构建我们的网站，难度不大。

开发过程

  1. 根据老师提供的链接，下载MDB的模板
  2. 先对index.html进行创作，主要使用到导航条，对本页面的品种进行了页面滑动跳转，对注意事项和实验报告进行页面跳转，这部分难度不大，在官网教程中可以轻松找到代码。
  3. 在对页面的主要内容进行创作的时候，我用到了栅格和卡片标签，并且添加了可以跳转的按钮。在style.css中添加背景图片。
  4. 在对不同的狗狗的主页面中，我用到了一个轮播的图片，部分我还添加了video用来展示视频。为他们的背景图添加了动态效果
  5. 在对网页的底部进行创作时，我采用的是简单的文字

遇到的已解决问题

  1. 设计页面平滑跳转的时候，跳转后导航条会把部分内容挡住

    2. 解决方法：我在每个div的前面加了一个pre标签，并且为pre标签添上了id，以此将内容与浏览器顶端空出导航条的位置

  2. 在放入视频的时候，不支持视频的格式

    3. 解决方法：用软件将下载的视频进行格式转换

  3. 在刚开始编辑页面的时候，写完导航条后，直接在main中写入自己的主要内容，这导致header中有部分留白

    4. 解决方法：在导航条后增加一定内容来填白，或者直接在header中编辑自己的主要内容（不推荐）

遇到的未解决问题

  1. 导入的背景图片未能全部显示
  2. 页面直接跳转和在本页面滑动跳转的导航条内容不能排列整齐

总结

本次课程结业实验，我学会了简单的网页Web制作，通过用改写MDB模板，来实现网页的制作。 实践过程中，我掌握了许多有关网页制作的知识，知道了静态网页和动态网页的区别。 在这次大作业中最让我印象深刻的是，当前我们浏览器的搜索器十分的拉跨，我想将视频转换格式，百度一下全是广告，极大的影响办事效率。 通过对静态网站的制作，我深深的明白了他的缺点。对于他的修改，相当的麻烦，耗时间。给我的观感来说，他十分的笨重，对于网页上的一些数据不能进行实时的更新。
